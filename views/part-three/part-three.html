All computer applications need to store and retrieve information. While a process is running, it can store a limited amount of information within its own address space.

However, the storage capacity is restricted to the size of the virtual address space. For some applications this size is adequate, but for others it's too small.

A second problem with keeping information within a processâ€™ address space is that when the process terminates, the information is lost.

A third problem is that it is frequently necessary for multiple processes to access (parts of) the information at the same time.

Thus, we have three essential requirements for long-term information storage:

  1. It must be possible to store a very large amount of information.
  2. The information must survive the termination of the process using it.
  3. Multiple processes must be able to access the information at once.

Just as the OS abstracts away:

  • the concept of the processor to create the abstraction of a process

  • the concept of physical memory to offer processes (virtual) address spaces,

we can solve these problems with a new abstraction: the file.
